Core Architecture — Multiplayer-First (MFA)

The entire game is coded as if it were a networked game with a server predicting local state. This means:

  • Simulation layer (pure GDScript Nodes — no rendering, no scene tree dependencies)
  • View layer (Godot scenes, TileMaps, UI) reads state but never writes it
  • Input → Command → Simulation → View — all state changes go through this pipeline
  • All state is JSON-serializable (no Vector2, no Node references in state dictionaries)
  • Fixed 20 Hz timestep — game logic is independent of render framerate

The heartbeat. Runs 20 Hz fixed timestep via _process(delta) with an accumulator. Speed multipliers scale wall-clock time consumed per tick.

_advance_tick() flow

current_tick++ → CommandQueue.dequeue_all() → for cmd: GameState.apply_command(cmd) → GameState.simulate_tick(current_tick) → EventBus.simulation_tick.emit(current_tick)
Why: All game logic runs here, not in _process. delta is accumulated; no logic reads delta directly. Deterministic at any speed.

InputMapper + InputSetup simulation/core/InputMapper.gd

InputMapper is the ONLY file in the codebase that calls Godot's Input API. It translates physical events into Command dicts and enqueues them. It is NOT an autoload — it attaches to a View scene node.

InputSetup is an autoload that registers all input actions programmatically (avoiding Godot's verbose project.godot event serialization).

PlacementValidator simulation/buildings/PlacementValidator.gd

Validates PLACE_BUILDING commands before any state mutation. Pure functions — never mutates state except deduct_cost().

Validation Pipeline (in order)

1. is_valid_type → INVALID_TYPE 2. bounds check (all footprint tiles) → OUT_OF_BOUNDS 3. occupancy check (all footprint tiles) → OCCUPIED 4. terrain bitmask check (all footprint tiles) → WRONG_TERRAIN 5. tech requirements → MISSING_TECH 6. resource / gold cost → MISSING_RESOURCES 7. shire influence radius (if shire_id >= 0) → OUTSIDE_BORDERS

ValidationResult Enum

OK=0 OUT_OF_BOUNDS=1 OCCUPIED=2 WRONG_TERRAIN=3 MISSING_TECH=4 MISSING_RESOURCES=5 OUTSIDE_BORDERS=6 INVALID_TYPE=7
Multi-tile buildings (e.g. 2×2 stone_quarry) check ALL tiles against terrain_req. Setting only the origin tile to ROCK is not enough — all 4 tiles must be ROCK.
